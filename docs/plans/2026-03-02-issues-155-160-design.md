# Issues #155–#160 Design

## Goal

Fix PutzOrga UI spacing, improve calendar colors/scope-selector/vacation display, create extended test data, and add SuperAdmin user impersonation.

## Issue #155 — PutzOrga UI Verbesserungen

**Problem:** Poor spacing, tight margins, missing visual separators in cleaning views.

**Fix:** CSS/template polish only — no component refactoring.

- **CleaningView.vue:** Header margin 0.5rem → 1rem, mobile card padding, h3 margin reset
- **CleaningSlotView.vue:** Back button mb-1 → mb-3, section divider before registrations table, action button grouping
- **AdminCleaning.vue:** Replace `p-4` utility with scoped CSS, narrow mobile media query scope

## Issue #160 — Userwechsel (Impersonation)

**Problem:** SuperAdmin cannot test the system as other users during dev/test phase.

**Fix:**

Backend:
- `POST /api/v1/auth/impersonate` — SUPERADMIN only, requires `impersonation` module toggle
- Body: `{targetUserId}`. Returns JWT with `impersonatedBy` claim containing admin's userId
- `POST /api/v1/auth/stop-impersonation` — reads `impersonatedBy` from current token, generates fresh admin JWT
- JwtAuthenticationFilter: allow `impersonatedBy` claim (no `type` claim blocking)
- Audit-log each impersonation start

Frontend:
- `ImpersonationBanner.vue` — red banner "Angemeldet als {Name}" with "Zurück" button, mounted in App.vue
- Auth store: read `impersonatedBy` from JWT claims
- Admin Settings (Sicherheit group): red-highlighted toggle with danger warning

DB:
- Flyway migration: add `impersonation: false` to modules JSONB
